# commit-msg examples

## 大原則 — WHY を書く、HOW は書かない

**Bad（実装の HOW を箇条書きで羅列している）**
```
feat: リハ患者一覧で起動時にメッセージ受信ウィンドウを表示

- スタッフ設定の会計／検査結果／スタッフメッセージのいずれかが有効な場合に起動時表示
- FormFactory 経由で MessageReceiver を生成し、Owner を自フォームに設定
- フォームクローズ時に MessageReceiver を明示的に解放
- using 並びと msg フラグの条件式を整理し、メソッド名を PascalCase に統一
```

問題点：
- 「FormFactory 経由で生成」「Owner を自フォームに設定」「Close で解放」は **diff を見れば分かる実装の HOW**
- 「using 並び」「メソッド名を PascalCase」は機械的なリファクタの羅列。diff の責務
- subject の言い換えで容量を浪費している

**Good（WHY と影響だけを書く）**
```
feat: リハ患者一覧で起動時にメッセージ受信ウィンドウを表示する

スタッフが起動直後に会計・検査結果・スタッフメッセージの通知を
見落とさないよう、リハ患者一覧の表示時に受信ウィンドウを自動で開く。

- 上記いずれの通知種別もスタッフ設定で無効化されている場合は表示しない
```

ポイント：
- 本文段落 = **WHY**（業務上の動機）
- 箇条書き 1 件 = **影響/前提**（レビュアーが気にする条件分岐）
- 実装 HOW は完全に省略 — diff と source を読めば分かる

## stage モード — シンプルなケース（body 省略）

**コマンド**
```
/commit-msg stage
```

**出力例**
```
fix: 商品名の表示で改行コードが二重に出ていた不具合を修正
```

`subject` だけで WHAT も WHY も自明。**body は無理に書かない**。

---

## stage モード — context（チケット番号）付き

**コマンド**
```
/commit-msg stage ABC-456
```

**出力例**
```
fix: カート削除時の NullPointerException を解消する (ABC-456)

空カートでも削除 API が呼ばれる経路があり、本番でクラッシュ報告があった。
```

---

## mode 省略 — staged 差分から自動判定

**コマンド**
```
/commit-msg
```

**前提**：staged 差分あり、ブランチ名が `feature/ABC-789-add-export`

**出力例**
```
feat: 顧客一覧画面に CSV エクスポート機能を追加する (ABC-789)

営業チームから手元集計の効率化要望があり、現在の絞り込み条件をそのまま
エクスポートできるようにする。大量データを扱うためストリーミングで出力する。

- 出力対象カラムは設定ファイルで切り替え可能
```

---

## latest モード — 直近コミットを集約

**コマンド**
```
/commit-msg latest
```

**出力例**
```
refactor: 認証ミドルウェアを共通クラスに集約する

各ルートに散在していた認証ロジックを 1 箇所で保守できるようにする。
今後の SSO 移行に向けた前段整理。

- 既存エンドポイントの挙動には変更なし
```

---

## latest モード — context で背景を補足

**コマンド**
```
/commit-msg latest セキュリティ監査での指摘対応
```

**出力例**
```
fix: セッショントークンの保存方式を HttpOnly Cookie に変更する

セキュリティ監査で localStorage 利用を指摘されたため、XSS 経由で
トークンが流出するリスクを排除する。

- 既存トークンは初回ログイン時に自動で新方式へ移行する
- CSRF 対策として SameSite=Lax を付与
```

---

## commit モード — 特定コミットIDから生成

**コマンド**
```
/commit-msg commit a1b2c3d
```

**出力例**
```
docs: API 仕様書にエラーコード一覧を追記する

クライアント実装側からエラーハンドリング指針が不明という問い合わせが
複数あったため、想定する応答コードを仕様に明示する。
```

---

## chore — ライブラリ更新（body 省略）

**コマンド**
```
/commit-msg stage
```

**出力例**
```
chore: React を 18.3 にアップグレードする
```

依存更新は subject だけで十分。**非推奨 API 置換などの実装変更が混ざる場合は別コミット**にする。

---

## perf — パフォーマンス改善（影響を数値で示す）

**コマンド**
```
/commit-msg stage
```

**出力例**
```
perf: 商品検索 API のレスポンスを高速化する

ピーク時間帯にタイムアウトが頻発していたため、N+1 を解消する。

- 平均応答時間が約 65% 短縮（実測 800ms → 280ms）
- 既存の検索結果順序は維持
```

## ci — CI/CD設定の変更

**コマンド**
```
/commit-msg stage
```

**出力例**
```
ci: PR 作成時にユニットテストを自動実行する

レビュー前にローカル実行漏れで CI が main で初めて落ちる事故が
続いたため、PR の段階で必ず実行されるようにする。

- Node.js 18 / 20 のマトリクスで実行
```

## 複数目的の混在 — feat を優先 + 分割提案

**コマンド**
```
/commit-msg stage
```

**出力例**
```
feat: お気に入り機能を追加する

商品比較フローの利便性向上のため、ユーザーごとに任意の商品を
保存しておけるようにする。

- 未ログイン時はローカルストレージに退避し、ログイン後に統合する

<!-- 独立した変更が含まれています。分割コミットを検討してください -->
```
