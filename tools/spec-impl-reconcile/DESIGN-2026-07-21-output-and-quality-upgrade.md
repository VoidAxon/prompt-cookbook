# spec-impl-reconcile 改修設計 — 出力刷新 + 品質ゲート強化（2026-07-21）

## 背景・動機
本スキル（A）の強み＝多エージェント分節並行の網羅性・持続的な成果物・子画面実在確認の厳密さ・非開発者向けの可読性。
姉妹的な公開スキル phase2-spec-review（B）の強み＝実在ゲート（ハルシネーション検出）・アドバサリアル収束ゲート・人確認段・見落観点集。
**両者を結合**し、A の「広さ＋成果物」に B の「確度＋幻覚防御」を接ぐ。さらに業界調査（RTM／documentation drift／LLMハルシネーション検出／Reversa 等）を踏まえ **確度タグ** と **阻断性Gap区分** を追加する。**A/B は無関係な別スキル。本改修は A（本リポジトリの spec-impl-reconcile）のみを対象とする。**

## 不変の定位
- 目的：設計書 × 実装の突合、要件整理（記載漏れ・相違・要確認）。**バグ摘出しない。read-only。**
- **領域・言語非依存**：医療安全・C#/WinForms 等は「例」に降格。医療安全 → 一般化した「高リスク領域」の一実例。

## 変更点
1. **出力を ①結論 / ②1指摘=1ブロック / ③転記用 / ④人確認段 に刷新**（詳細 references/output-format.md）。
2. **③ 列（9列）**：`課題ID | 対象 | 指摘箇所(§見出し) | 指摘内容 | 分類 | 深刻度 | 確度 | 対処区分 | 備考`。下拉許可値・台帳IDはハードコードしない。
3. **format パラメータ** `md|excel|both`（既定 md）。md=①〜④ を .md、excel=③ を真xlsx（openpyxl→ImportExcel→CSV降級・告警）。
4. **実在ゲート**を導入（核心シンボル0件＝B・高・再抽出、真の実体を帰属先の誤りとして記録）。
5. **アドバサリアル収束ゲート**（出力前必須・loop-until-stable）。
6. **人確認段 + 見落観点集**（一般原則として内蔵）。
7. **確度タグ Confirmed/Inferred/Gap**（各指摘に必須。Inferred/Gap は理由明記）。
8. **阻断性Gap区分**（④で 阻断性Gap／非阻断Gap を分離）。

## 保持
多エージェント分節並行（子agent Write + 総括）／子画面実在確認／分類 A1–E／深刻度・対処区分の既定対応／出力先 `%USERPROFILE%\Documents\<対象名>\`。

## 強化（2026-07-21 phase2 との比較で判明 — 両者の長所を集約）
ReceMain の実データで phase2-spec-review（インライン）と本スキル（fan-out）を比較した結果、両者の漏れは**非対称**だった。インラインは"コードを辿るだけ"で構造カバレッジ（子画面・トリガー・サーバーコントロール一覧）を得るが深掘り長尾を落とし、fan-out は深掘りに強いが分節の切り方・種子偏りで構造盲点が出る。両者の長所を集約するため次を追加：
- **機械枚举母集団突合（§4.0）**：機械で全量枚举できる5母集団（子画面/トリガー配線/サーバーコントロール・データ源/設定キー/一覧の列）を必ず全量枚举して仕様書と**双方向**突合（表に無い実装＝欠落／実装に無い表行＝死配線）。fan-out にインライン相当の構造カバレッジを与える。
- **経路差レンズ（§4.1）**：新規/更新/取消で発火処理が違う点。
- **完整性クリティック（§6.5）**：出力前に「5母集団を全量突合したか・分節の隙間・未読引用ソース」を能動確認（silent cap 禁止）。兼顾できない場合も母集団チェックリストが下限（バランス）を担保。
- subagent-prompt に「構造カバレッジ担当」を必ず1つ立てるレンズ配分を明記。

## 検証（2026-07-21 実施）
writing-skills の RED-GREEN をサブエージェントで実施（fixture＝幻覚Service層＋横断欠落を含むAI生成仕様書＋実コード）。
- **RED（skill無し）**：幻覚・横断欠落は検出できたが、①〜④構造・確度・分類・スコープ規律が欠落。空catchをバグとして起票推奨（バグ狩猎へ漂流）、既知バグ候補B-01を重複再報。
- **GREEN（skill有り）**：実在ゲートで幻覚を B・高・再抽出、横断欠落を A2・高・【高リスク】、①〜④＋確度＋阻断性Gap、B-01は確認済みメモに留め重複回避、収束ゲートで過大主張を調整。**合格**。
- **REFACTOR-1**：GREEN が③の機械チェックを自己申告で済ませ末尾パイプが混入。`pipe_to_xlsx.py` が exit 2 で検出することを実証し、output-format.md と SKILL.md に「format に関わらずスクリプトを必ず実行し exit 0 を確認してから③確定・自己申告禁止」を追記。
- **REFACTOR-2（2026-07-21 実データ ReceMain で判明）**：分節並行で child-forms 分節に狭い候補リストを渡したところ、グリッドの主入力子画面 `ReceSub`（`_OpenReceSub`）を丸ごと見落とした（phase2-spec-review はインライン読解で捕捉＝P-01）。原因は「候補リスト依存」。review-method.md §2 と subagent-prompt.md の子画面レンズに「起動パターン(`new .*Form`/`ShowDialog`/`ShowModal`/`_Open`系/`KickFunction`/`FormFactory.`)を grep で機械枚举してから実在確認・候補リストに依存しない」を追記。修正後の子画面レンズを ReceMain で単独再実行し、ReceSub を最優先 A1 として検出できることを確認（loophole closed）。教訓＝機械枚举できるものは候補リスト/自己申告に頼らない（REFACTOR-1 と同型）。
