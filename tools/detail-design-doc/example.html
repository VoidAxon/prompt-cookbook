1. 概要

1.1 機能の目的

選択中の患者について、過去の検査結果を期間・検査種別で絞り込んで一覧表示し、行を選択して結果詳細画面を開くための照会画面。参照専用であり、本画面から検査データの登録・修正は行わない。

1.2 スコープ

  • 対象：検索条件の入力、結果一覧の表示、詳細画面への遷移、異常値の強調表示
  • 対象外：検査オーダの発行（→ DD-ORD-0005）、結果詳細の表示内容（→ DD-LAB-0022）

1.3 前提条件・制約

  • カルテ画面で患者が選択済みであること（患者未選択では本画面を開けない）
  • 参照専用。本画面からのDB更新は一切行わない
  • 表示対象は確定済み結果のみ。未確定（速報）結果は設定（§8）により表示可否が切り替わる

1.4 用語定義

用語 定義
基準値外 検査項目ごとに定義された基準範囲（下限〜上限）を外れた結果値
速報 検査部門で最終確定前に送信される暫定結果

5.2 処理詳細

P-02: 結果検索

項目 内容
契機 btnSearch の Click
入力 §4.1(1) の検索パラメータ
出力 / 画面反映 結果一覧の再表示、件数の表示
前提条件 入力チェック（§3.3）を通過していること
事後条件 「詳細」Button は無効に戻る（行選択が解除されるため）

処理の流れ

  1. 入力チェック（§3.3）を行う。NG の場合はメッセージを表示し中断する
  2. 検索パラメータを組み立てる。検査種別が「全て」の場合、種別条件は付けない。設定〈LabResult.ShowPreliminary〉が ON のとき、確定区分の絞り込みを外し速報も対象に含める
  3. サーバへ検索を依頼し、結果を受け取る（SQL は §6.2 Q-01）
  4. 各行の判定列を §5.3 の条件→結果表に従い設定する
  5. グリッドへバインドし、初期ソートを適用する
  6. 0件・上限超過の場合は §4.1(4) のとおりメッセージを表示する

P-03: 詳細表示

項目 内容
契機 dgvResults の行ダブルクリック、または btnDetail の Click
入力 選択行の検査依頼番号（非表示の返却項目 No.1）
出力 / 画面反映 検査結果詳細画面をモーダルで表示
前提条件 行が1行選択されていること
事後条件 詳細画面を閉じた後、一覧の選択状態・スクロール位置は保持される

処理の流れ

  1. 選択行から検査依頼番号を取得する
  2. 検査結果詳細画面に検査依頼番号・患者IDを渡して開く

6.4 SQL定義（現行挙動の参考）

-- Q-01: 検査結果一覧検索（対応 UC: UC-03）
SELECT
    r.request_no,
    r.test_date,
    c.category_name,
    i.item_name,
    r.result_value,
    i.unit,
    i.ref_low,
    i.ref_high,
    r.abnormal_flag,
    r.confirm_status
FROM t_lab_result r
    INNER JOIN m_test_item     i ON i.item_code     = r.item_code
    INNER JOIN m_test_category c ON c.category_code = i.category_code
WHERE r.patient_id = @patientId
  AND r.test_date BETWEEN @dateFrom AND @dateTo
  /* 種別「全て」以外のとき */ AND i.category_code = @categoryCode
  /* 基準値外のみ ON のとき */ AND r.abnormal_flag = 1
  /* 速報表示 OFF のとき */    AND r.confirm_status = '1'  -- 1:確定
ORDER BY r.test_date DESC, i.display_order ASC
項目 内容
対応 UC UC-03
実行経路 検索 Button → サーバコマンド（検査結果一覧取得）
想定件数 / 性能上の注意 患者・2年範囲で最大数千件。上限2,000件で打切り

※ 本 SQL は現行実装の記録であり、移行先が結合構造・打切り方式を踏襲する契約ではない。契約は UC-03 の定義。

10. 医療安全上の留意点

  • 患者識別: 本画面は呼出元から渡された患者IDに固定され、画面内で患者を切り替えられない。表示中も患者ID・氏名を常時表示し、別患者の結果と取り違えない構造とする。
  • 速報結果の扱い: 速報（未確定）結果は確定値と異なる可能性があるため、既定では表示しない。設定で表示する場合も「速報」の明示とグレー表示で確定結果と視覚的に区別する（§4.1(2)）。
  • 基準値判定の限界: 判定列は基準値マスタに依存する。マスタ未設定の項目は判定しない（空欄）仕様であり、「空欄＝正常」ではないことを運用上周知する必要がある。
  • 表示の打切り: 件数上限で打切った場合は必ず警告（M-04）を出し、全件表示と誤認させない。
  • 〔推定〕速報を既定で非表示にしているのは、確定前の暫定値を確定値と取り違えさせないための設計と考えられる（既定値 false は §8 で確認済みだが、その意図を明示するコメントはコード上に無いため推定）。
